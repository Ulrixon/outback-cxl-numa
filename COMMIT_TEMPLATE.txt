feat: Add NUMA-based memory disaggregation support alongside RDMA

Add NUMA support as an alternative to RDMA for memory disaggregation,
enabling OUTBACK to run on single machines with multiple NUMA nodes
without requiring RDMA hardware. Original RDMA functionality remains
fully intact and unchanged.

## Motivation

- Enable testing of memory disaggregation concepts without RDMA NICs
- Simplify development and debugging on standard hardware
- Support CXL and memory tiering research scenarios
- Reduce setup complexity for evaluation (10min vs 1hr)
- Provide lower-latency local disaggregation option (~100ns vs ~1µs)

### Key Technical Details

NUMA Architecture:
- Client (CN): Runs on local DRAM (NUMA node 0), maintains index cache
- Server (MN): Data structures allocated on remote NUMA node 1
- Communication: Direct memory access via load/store instructions
- No network stack, no RDMA verbs, no RPC serialization

Data Flow:
1. Client calculates slot location using local ludo_lookup_unit
2. Client directly reads ludo_buckets on remote NUMA node
3. Client directly reads/writes packed_data on remote NUMA node
4. Synchronization via mutex arrays on NUMA node

## Usage

### NUMA Setup (New)

Setup:
```bash
./setup-env.sh numa            # Install deps, no RDMA/OFED
./build_numa.sh                # Build NUMA version
```

Run:
```bash
# Server (terminal 1)
sudo ./build/benchs/outback/server_numa --numa_node=1 --nkeys=10000000 --seconds=60

# Client (terminal 2)
sudo taskset -c 0-7 ./build/benchs/outback/client_numa --nkeys=10000000 --threads=8 --seconds=60
```

### RDMA Setup (Original - Unchanged)

Setup:
```bash
./setup-env.sh                 # Install deps + OFED
./build.sh                     # Build RDMA version
```

Run:
```bash
# Server machine
sudo ./build/benchs/outback/server --nkeys=50000000 --seconds=120

# Client machine
sudo ./build/benchs/outback/client --server_addr=192.168.1.2:8888 --nkeys=50000000 --threads=8
```

## Comparison

|--------------------|-------------------|------------------|
| Hardware           | Standard NUMA     | RDMA NICs        |
| Setup Time         | 10-15 min         | 30-60 min        |
| Deployment         | Single machine    | Multiple machines|
| Latency            | ~100-300ns        | ~1-5µs           |
| Network Required   | No                | Yes              |
| OFED Required      | No                | Yes              |
| Use Case           | Dev/Test/CXL      | Production       |

- RDMA-capable NICs
- Network connectivity
